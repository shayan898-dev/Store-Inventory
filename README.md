# ЁЯПк Inventory Manager Pro

A modern, dark-themed desktop inventory management application for small retail stores, built with Python, CustomTkinter, and SQLite.

---

## тЬи Features

| Feature | Description |
|---|---|
| **Inventory Management** | Add, edit, delete, and search products by barcode, name, or category |
| **High-Speed Sales Mode** | Barcode-scanner-optimised checkout interface with ghost-focus loop |
| **Restock Mode** | Scan a barcode, enter units to add тАФ log is auto-persisted |
| **Reports Dashboard** | Live stats: total products, total items, low-stock alerts, inventory value |
| **Category Support** | Products grouped into customisable categories with a dropdown + free-text entry |
| **Low-Stock Alerts** | Rows highlighted amber when quantity falls below 5; dedicated Reports list |
| **CSV Export** | One-click export of full inventory to a timestamped `.csv` file |
| **Persistent Logs** | Last 50 sales and last 50 restocks stored in SQLite across sessions |
| **Schema Migration** | Safe `ALTER TABLE` migration тАФ existing databases gain new columns without data loss |
| **Standalone EXE** | Ships as a single `.exe` via PyInstaller тАФ no Python install needed on target machine |

## ЁЯЧВ File Analysis

### `main.py` тАФ GUI Application

The main entry point. Launches a `customtkinter` window (`InventoryApp`) with four views accessible from a sidebar:

#### Views

| View | Purpose |
|---|---|
| **ЁЯУж Inventory** | CRUD table with checkbox multi-select, search, CSV export |
| **ЁЯЫТ Sales** | Barcode scan тЖТ deduct stock by 1, flash feedback, live transaction log |
| **ЁЯУе Restock** | Barcode scan тЖТ pop-up quantity dialog тЖТ add stock, log persisted |
| **ЁЯУК Reports** | Summary stat cards + scrollable low-stock product list |

#### Key Classes & Helpers

| Symbol | Kind | Description |
|---|---|---|
| `InventoryApp` | class | Root `CTk` window; owns all views and DB instance |
| `SidebarButton` | class | Custom `CTkButton` with active/inactive state styling |
| `StatCard` | class | Dashboard tile showing icon, value, and label |
| `FlashEntry` | class | `CTkEntry` that briefly flashes its border green/red on scan result |
| `ProductDialog` | class | Modal `CTkToplevel` for Add / Edit product |
| `RestockQtyDialog` | class | Small modal asking how many units to restock |
| `format_price(v)` | function | Formats a float as `PKR 1,234.00` |
| `play_beep(success)` | function | 1 kHz/150 ms success beep or 400 Hz/400 ms failure buzz (`winsound`) |
| `ts_now()` | function | Returns current time as `HH:MM:SS` string |
| `_sanitise_barcode(raw)` | function | Strips whitespace and non-printable ASCII characters from raw scanner input |

#### Ghost Focus Loop (`_scan_focus_tick`)

Every 500 ms, while in Sales or Restock view, the app checks whether a modal dialog is open. If none is open and the barcode entry does not already have focus, it is re-focused automatically тАФ allowing 50+ items to be scanned without touching the keyboard or mouse.

### `database_manager.py` тАФ SQLite Layer

All database access is encapsulated in `DatabaseManager`. A `threading.Lock` serialises write operations; WAL journal mode is enabled for better concurrent read performance.

#### Tables

| Table | Columns | Description |
|---|---|---|
| `products` | `barcode` (PK), `name`, `category`, `quantity`, `price`, `timestamp` | Master inventory |
| `sales_log` | `id`, `barcode`, `name`, `quantity_sold`, `remaining_qty`, `timestamp` | Last 50 sale events |
| `restock_log` | `id`, `barcode`, `name`, `quantity_added`, `remaining_qty`, `timestamp` | Last 50 restock events |

#### Public API

```python
db = DatabaseManager()

# тФАтФА Products тАФ Read тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФА
db.get_all_products()                       # тЖТ list[dict]  (sorted by name)
db.search_products(query)                   # тЖТ list[dict]  (barcode / name / category LIKE)
db.find_by_barcode(barcode)                 # тЖТ dict | None
db.get_low_stock_products(threshold=5)      # тЖТ list[dict]  (qty < threshold)
db.get_summary_stats()                      # тЖТ dict  {total_products, total_items, low_stock_count, total_value}
db.get_categories()                         # тЖТ list[str]   (distinct + built-in defaults)

# тФАтФА Products тАФ Write тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФА
db.add_product(barcode, name, category, quantity, price)    # тЖТ bool
db.update_product(barcode, name, category, quantity, price) # тЖТ bool
db.update_stock(barcode, amount)            # тЖТ ("ok"|"not_found"|"out_of_stock", dict|None)
db.delete_product(barcode)                  # тЖТ bool

# тФАтФА Logs тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФА
db.log_sale(barcode, name, quantity_sold, remaining_qty)
db.log_restock(barcode, name, quantity_added, remaining_qty)
db.get_recent_sales(limit=20)               # тЖТ list[dict]  newest first
db.get_recent_restocks(limit=20)            # тЖТ list[dict]  newest first

# тФАтФА Export тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФА
db.export_to_csv(filepath)                  # тЖТ bool
```

`update_stock` is thread-safe and handles three outcomes:

| Return status | Meaning |
|---|---|
| `"ok"` | Stock changed successfully; updated product dict returned |
| `"not_found"` | Barcode not in the database |
| `"out_of_stock"` | `amount` is negative but `quantity` is already 0 |

### `build_app.py` тАФ PyInstaller Build Script

Cleans old `build/` and `dist/` folders, then invokes PyInstaller with:

| Flag | Effect |
|---|---|
| `--onefile` | Single self-contained `.exe` |
| `--noconsole` | No black terminal window |
| `--noupx` | UPX compression disabled (avoids antivirus false positives) |
| `--noconfirm` | Overwrites `dist/` without prompting |
| `--add-data database_manager.py:.` | Bundles the DB module alongside the entry point |

Run with:
```bash
python build_app.py
```

Output: `dist/InventoryManager.exe`

### `InventoryManager.spec` тАФ PyInstaller Spec

Auto-generated spec file used by PyInstaller. Key settings:
- Entry point: `main.py`
- Data files: `database_manager.py` bundled into the root of the archive
- Console: disabled (`console=False`)
- UPX: disabled (`upx=False`)

Can be used directly with `pyinstaller InventoryManager.spec` instead of re-running `build_app.py`.

### `inventory.db` тАФ SQLite Database

Auto-created next to the script (or `.exe`) on first run. Contains three tables: `products`, `sales_log`, and `restock_log`. The database supports safe schema migration тАФ older databases are upgraded automatically without data loss.

## ЁЯЪА Getting Started

### Prerequisites

- Python 3.10 or later
- Windows (recommended тАФ `winsound` beep is Windows-only; the app falls back to a terminal bell on other platforms)

### Install dependencies

```bash
pip install -r requirements.txt
```

### Run in development

```bash
python main.py
```

### Build standalone EXE (Windows)

```bash
python build_app.py
```

The compiled binary is placed at `dist/InventoryManager.exe`. Copy it anywhere тАФ no Python installation is required on the target machine. `inventory.db` is created automatically next to the `.exe` on first launch.

## ЁЯУК Database Schema

```sql
CREATE TABLE products (
    barcode   TEXT PRIMARY KEY,
    name      TEXT NOT NULL,
    category  TEXT NOT NULL DEFAULT 'General',
    quantity  INTEGER NOT NULL DEFAULT 0,
    price     REAL NOT NULL DEFAULT 0.0,
    timestamp TEXT NOT NULL DEFAULT (strftime('%Y-%m-%d %H:%M:%S','now','localtime'))
);

CREATE TABLE sales_log (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    barcode       TEXT NOT NULL,
    name          TEXT NOT NULL,
    quantity_sold INTEGER NOT NULL,
    remaining_qty INTEGER NOT NULL,
    timestamp     TEXT NOT NULL DEFAULT (strftime('%Y-%m-%d %H:%M:%S','now','localtime'))
);

CREATE TABLE restock_log (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    barcode         TEXT NOT NULL,
    name            TEXT NOT NULL,
    quantity_added  INTEGER NOT NULL,
    remaining_qty   INTEGER NOT NULL,
    timestamp       TEXT NOT NULL DEFAULT (strftime('%Y-%m-%d %H:%M:%S','now','localtime'))
);
```
